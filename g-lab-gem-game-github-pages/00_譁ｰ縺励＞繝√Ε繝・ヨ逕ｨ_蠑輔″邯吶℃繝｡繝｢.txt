【花の仕様】
・花は「むつかしい」モードだけに出す
・出現確率は約1/30、上記5種からランダム
・花は種の状態で落下し、最初の着地後に開花する
・開花時はハープ系の「ポロン」音
・花は消えない、合体しない、花同士が重なっても消えない
・影なし、背景透過、輪郭に沿ってトリミング

【花の大きさ】
・梅：アクアマリン基準の12%小さめ（scale 0.88）
・芍薬：ダイヤモンド基準
・菊：エメラルド基準
・牡丹：真珠基準
・斑入り牡丹：牡丹と同じ

【難易度】
・やさしい：ガーネット、アメシスト、アクアマリンは出ない。花は出ない。すべての宝石をふつう・むつかしいより5%小さくする。
・ふつう：通常設定。花は出ない。
・むつかしい：宝石サイズ・重なり方はふつうと同じ。花のみ追加。

【その他の確定仕様】
・COMBOの文字表示は出さない（コンボ得点倍率は残してよい）
・DANGER LINE表示は出さない
・宝石または花が天井より半分以上出た状態が1.5秒続くとゲームオーバー
・リセットボタンは不要
・「今の宝石」「次の宝石」は縦を小さめ
・BGMは従来より約20%大きめ、通常の効果音は維持
・操作盤面の背景は assets/board-bg-v89.png
・盤面中央には薄い g-Lab. ロゴを残す
・アプリアイコンは現行の icons フォルダ内の4ファイルを使う

【キャッシュ対策】
・service-worker.js の CACHE_NAME は更新ごとに必ず変更する
・index.html の app.js クエリ（例：app.js?v=89）も更新する
・花画像・盤面背景のパスは新しいバージョン名を付けて参照し、古いキャッシュと混ざらないようにする
・更新用ZIPには、変更した画像を必ず含める

【更新後の確認】
1. GitHub Desktopで変更一覧に app.js / index.html / service-worker.js / assets内の画像が表示されることを確認
2. Commit to main → Push origin
3. 数分待って公開ページを再読み込み
4. 必要ならブラウザで強制再読み込み
5. PWAをホーム画面に追加している場合は、一度削除して再追加

【作業ルール】
ユーザーが複数の花画像を送る場合、ユーザーが「ゲームへ反映してください」と言うまでゲームデータを作らず、変更予定のみ記録する。
反映指示後は、必ず「ゲーム全体ZIP」と「GitHub更新用ZIP」の両方を作る。
作成前に app.js の花一覧、画像パス、service-worker.js のキャッシュ名、更新用ZIPの内容を確認する。
